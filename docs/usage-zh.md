## 1. 界面导航

### 手机端（竖屏）

底部导航栏包含 5 个主入口：

| 图标 | 页面 | 说明 |
|---|---|---|
| 🏠 | **仪表盘** | 实时数据总览 |
| 📱 | **设备** | 已知设备列表 |
| 🔔 | **告警** | 活跃告警列表 |
| 📤 | **发送** | 向节点发送指令 |
| ⚙️ | **设置** | 连接、主题、语言配置 |

点击左上角 **☰ 菜单** 打开侧边抽屉，可访问：
- 🗺️ 设备地图
- 📜 历史数据
- ⚙️ 规则引擎
- 💊 系统健康

### PC / 桌面端（宽屏）

左侧 **NavigationRail** 替代底部导航栏：

- 窗口宽度 **600–1199px**：紧凑轨道，仅显示图标
- 窗口宽度 **≥1200px**：扩展轨道，显示图标 + 标签
- 底部额外 4 个功能按钮（地图 / 历史 / 规则 / 健康）在 1200px 以下为 2×2 布局，1200px 以上为 1×4 单行

### 手机横屏

横屏时高度 ≤ 500px，自动切回底部导航栏布局，避免 NavigationRail 溢出。

## 2. 首次连接设置

进入 **设置 → 连接** 标签页：

### 2.1 UDP 数据接收（推荐）

```
启用 UDP 监听    ✅ 开启
监听地址         0.0.0.0（监听所有网卡）
监听端口         9876（需与节点固件一致）
```

**操作步骤：**
1. 确认手机/PC 与 OpenSynaptic 节点连接到同一 Wi-Fi / 局域网
2. 启用 UDP 监听开关
3. 如需修改端口，输入后点击 **保存连接设置**
4. 节点上电后即开始发送数据，仪表盘自动更新

> 💡 UDP 数据接收依赖 Wi-Fi 或以太网，蜂窝数据（4G/5G）通常不支持 UDP 广播。

### 2.2 MQTT 数据接收（可选）

```
启用 MQTT        ✅ 开启
Broker 地址      localhost（本地 MQTT Broker）或填写远程地址
Broker 端口      1883（标准 MQTT 端口）
```

同时启用 UDP 和 MQTT 时，两路数据均会显示在仪表盘，系统自动去重设备。

### 2.3 蓝牙 BT-UART（实验性）

> ⚠️ 此功能为实验性，需固件支持 BT-UART 透传模式。

1. 先在系统蓝牙设置中完成设备配对
2. 输入设备 MAC 地址（格式：`AA:BB:CC:DD:EE:FF`）
3. 填写虚拟端口号（默认 9877）
4. 保存后启用蓝牙传输开关

### 2.4 网络状态检测

**Wi-Fi / 网络状态** 卡片显示当前连接类型（Wi-Fi / 以太网 / 移动网络）。  
点击右侧 🔄 按钮手动刷新检测结果。

## 3. 实时仪表盘

仪表盘数据每 **5 秒**自动刷新一次，同时实时响应节点推送的传感器数据。

### 3.1 KPI 卡片

| 卡片 | 说明 |
|---|---|
| **总设备数** | 数据库中已知设备总数 |
| **在线率** | 在线设备 / 总设备，颜色代码：绿色 ≥90%，黄色 ≥70%，红色 <70% |
| **活跃告警** | 未确认告警数量，有告警时显示红色 |
| **消息吞吐量** | 当前每秒接收消息数（msg/s） |

> PC 端 KPI 卡片等宽横排，手机端 2×2 网格。

### 3.2 折线图

- **温度趋势**：实时滚动显示最近 60 条温度读数
- **湿度趋势**：实时滚动显示最近 60 条湿度读数
- 图表包含警告阈值线（黄色）和危险阈值线（红色）

> PC 端两图并排显示，手机端垂直堆叠。

### 3.3 仪表盘（Gauge）

| 仪表 | 量程 | 阈值 |
|---|---|---|
| 温度 | -20 ~ 80 °C | 警告 40°C，危险 60°C |
| 气压 | 900 ~ 1200 hPa | 警告 1050，危险 1100 |
| 液位 | 0 ~ 100% | - |
| 湿度 | 0 ~ 100% | 警告 80%，危险 95% |

> PC 端 4 个仪表盘单行显示，手机端两行各 2 个。

### 3.4 图表（Bar / Pie）

- **柱状图**：多设备传感器数值对比
- **饼图**：设备类型分布（传感器 / 执行器 / 网关 / 其他）

> PC 端柱状图（60%）+ 饼图（40%）并排，手机端垂直排列。

### 3.5 自定义显示卡片

**设置 → 卡片** 标签页可单独开关各卡片/图表，保存后立即生效。

## 4. 设备管理

**设备** 页面显示所有已知设备（自动从接收到的数据中注册）。

| 字段 | 说明 |
|---|---|
| 设备名称 | nodeId 字段，固件中设置 |
| AID | 设备地址 ID（固件分配的唯一标识符） |
| 传输类型 | UDP / MQTT / BT |
| 状态 | 在线（绿色）/ 离线（灰色） |
| 最新读数 | 最近一次接收到的传感器值 |
| 最后在线 | 最后收到数据的时间（s/m/h ago） |

- 搜索框：按名称或 AID 过滤
- 点击设备卡片查看详细读数

## 5. 告警中心

**告警** 页面列出所有未确认告警，按时间倒序排列。

### 告警级别

| 级别 | 颜色 | 触发条件示例 |
|---|---|---|
| **严重** (Critical) | 🔴 红色 | 温度超过危险阈值 |
| **警告** (Warning) | 🟡 黄色 | 传感器进入警告区间 |
| **信息** (Info) | 🔵 蓝色 | 规则引擎日志类触发 |

- 告警由规则引擎自动创建
- 点击告警卡片可查看触发设备和详细信息
- 告警同时触发本地通知（需授予通知权限）

## 6. 指令发送

**发送** 页面提供完整的 OpenSynaptic 指令发送能力，分 3 个标签页：

### 6.1 目标配置栏（顶部）

| 字段 | 说明 |
|---|---|
| 目标设备 / AID | 下拉选择已知设备，或手动输入 AID |
| TID | 事务 ID（0–255）|
| SEQ | 序列号（0–255）|
| 目标 IP | 节点 IP 地址（UDP 发送时使用）|
| 端口 | 节点监听端口（默认 9876）|

发送优先级：已连接 MQTT → UDP（如有目标 IP）

### 6.2 控制命令（Control）

| 命令 | 字节码 | 说明 |
|---|---|---|
| PING | `0x01` | 心跳探测 |
| PONG | `0x02` | 回复 PING |
| HANDSHAKE_ACK | `0x05` | 确认握手 |
| HANDSHAKE_NACK | `0x06` | 拒绝握手 |
| ID_REQUEST | `0x03` | 请求分配 AID |
| ID_ASSIGN | `0x04` | 向设备分配 AID（需填写目标 AID）|
| TIME_REQUEST | `0x07` | 请求时间戳 |
| SECURE_DICT_READY | `0x0A` | 通知安全字典就绪 |

### 6.3 数据命令（Data）

**单传感器 DATA_FULL**：
- 选择传感器 ID（TEMP / HUM / PRES 等）
- 选择状态码（U=正常 / A=告警 / W=警告 / D=危险）
- 选择单位
- 输入数值
- 点击 **发送单传感器**

**多传感器 DATA_FULL**：
- 点击 ➕ 添加传感器行（最多可添加任意数量）
- 每行独立配置 SID / 单位 / 数值 / 状态
- 点击 **发送多传感器**

### 6.4 原始 HEX（Raw）

输入十六进制字节串（支持带空格或不带空格）：

```
示例：01 00         → PING 帧
      09 00 01 00   → DATA_FULL 帧头
```

页面底部提供 **CMD 字节速查表** 供参考。

### 6.5 发送日志

每次发送后日志面板实时更新，显示：
- 时间戳
- 命令名称
- ✅ 成功 / ❌ 失败
- 目标地址和帧大小

## 7. 设备地图

**设备地图** 页面在 OpenStreetMap 上显示设备位置（需设备有 GPS 坐标数据）。

### 地图瓦片服务器配置

在 **设置 → 连接 → 地图瓦片服务器** 中选择：

| 预设 | 说明 |
|---|---|
| OpenStreetMap | 默认，免费公开瓦片 |
| CartoDB Light | 浅色地图风格 |
| CartoDB Dark | 深色地图风格 |
| Stadia Alidade Dark | 深色专业地图 |
| Custom | 自定义 URL 模板（`{z}/{x}/{y}.png`）|

> 💡 国内环境访问 OpenStreetMap 可能较慢，建议部署私有瓦片服务器或使用 CartoDB。

## 8. 历史数据与导出

**历史数据** 页面展示过去 **24 小时**内的传感器记录（最多 500 条）。

### 字段说明

| 字段 | 说明 |
|---|---|
| 时间戳 | 接收时间（ISO 8601 格式）|
| 设备 AID | 发送数据的设备标识符 |
| 传感器 ID | TEMP / HUM / PRES 等 |
| 数值 | 传感器读数 |
| 单位 | °C / %RH / hPa 等 |

### 导出 CSV

点击右上角 **⬇️** 按钮，文件保存至应用文档目录：

```
文件名：export_<时间戳>.csv
路径（Android）：/data/user/0/<包名>/files/
路径（Windows/Linux）：用户文档目录
```

CSV 格式示例：
```csv
时间戳,Device AID,传感器,数值,单位
2026-04-09T10:00:00.000,1,TEMP,25.3,°C
2026-04-09T10:00:00.000,1,HUM,60.1,%RH
```

## 9. 规则引擎

**规则引擎** 页面分两个标签：**规则** 和 **操作日志**。

### 9.1 创建规则

点击右下角 **+** 按钮新建规则：

| 字段 | 说明 | 示例 |
|---|---|---|
| 规则名称 | 自定义描述 | "高温告警" |
| 传感器 ID 过滤 | 留空匹配所有传感器 | TEMP |
| 设备 AID 过滤 | 留空匹配所有设备 | 1 |
| 运算符 | `>` / `<` / `>=` / `<=` / `==` | `>` |
| 阈值 | 触发判断的数值 | 60 |
| 冷却时间（秒） | 同一规则两次触发的最小间隔，防止刷屏 | 60 |
| 触发动作 | 见下表 | 创建告警 |

### 9.2 触发动作类型

| 动作 | 说明 |
|---|---|
| **创建告警** | 在告警中心生成一条新告警，同时触发本地通知 |
| **发送命令** | 向目标设备发送自定义命令（目标 AID 留空则发给触发设备）|
| **仅记录日志** | 不触发告警，仅写入操作日志 |

### 9.3 操作日志

**操作日志** 标签显示最近 100 条规则触发记录，包含：
- 触发时间
- 规则名称
- 触发设备 AID 和传感器值
- 执行动作结果

### 9.4 规则说明

- 规则按接收到每条传感器数据时实时评估
- 冷却时间内重复触发的事件**不会**再次执行动作
- 规则持久化存储于本地 SQLite 数据库

## 14. 常见问题

### Q：为什么仪表盘没有数据？

**排查步骤：**
1. 检查 **系统健康** 页面 UDP 状态是否为 `Connected`
2. 确认手机/PC 与节点在**同一局域网**（同一 Wi-Fi 下）
3. 检查节点固件目标 IP 和端口配置（UDP 端口默认 9876）
4. Android 设备：确认应用有网络权限
5. Windows 防火墙：确认 UDP 9876 端口入站规则已放行

---

### Q：为什么 MQTT 显示 Disconnected？

1. 确认 MQTT Broker（如 Mosquitto）已启动
2. 检查 Broker 地址填写是否正确（IP 或域名）
3. 检查端口（标准 1883，TLS 8883）
4. 防火墙是否开放 1883 端口

---

### Q：如何向已有节点发送 PING？

1. 进入 **发送** 页面
2. 在顶部输入**目标 IP** 和**端口**
3. 切换到 **控制** 标签页
4. 点击 **PING → 发送** 按钮
5. 查看底部**发送日志**确认结果

---

### Q：规则引擎触发但没有收到告警通知？

1. 确认手机已授予 App **通知权限**（Android 13+ 需单独授权）
2. 检查规则的**冷却时间**是否过长（如设置 3600s，则 1 小时内只触发一次）
3. 进入**操作日志**确认规则是否确实被触发

---

### Q：CSV 文件在哪里？

- **Android**：文件管理器 → 内部存储 → `Android/data/<包名>/files/`，或使用 ADB `adb pull`
- **Windows/Linux**：用户文档目录，导出成功后 SnackBar 会显示完整路径

---

### Q：浅色主题下文字看不清？

App 已针对深色/浅色主题自动适配文字颜色。如遇问题，请确认使用的是 v1.0.1 或更新版本（该版本修复了浅色主题下文字颜色问题）。
